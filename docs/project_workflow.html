Project Workflow

Overview

This document describes the full data processing, routing, and modelling workflow used in the accompanying paper. The pipeline:

  1. Processes trip data from the DECISIONS GPS-tracked travel diary survey

  2. Computes travel time and distance attributes using three routing approaches:

    • R5: multimodal routing engine by Conveyal

    • DO: dodgr routing R package with Ordnance Survey Multimodal Routing Network

    • GR: Google Routes API

  3. Evaluates the accuracy of these methods relative to observed trip distances and durations

  4. Constructs a revealed-preference (RP) dataset

  5. Estimates multinomial logit models using the Apollo package

All steps are implemented in R and should be run sequentially.

Reproducibility and data access

Full replication of this workflow requires access to several restricted or proprietary datasets:

  • DECISIONS survey data with GPS locations (not redistributable)
  • OS Multimodal Routing Network (OSMRN) (licensed)
  • Google Maps Routes API key (paid service)

Where inputs cannot be redistributed, their required formats, file paths, and processing assumptions are documented below.

1 Process DECISIONS trip data

1.1 Inputs

Manually extracted from the DECISIONS dataset:

  • trips tab → input/raw/trips.csv
  • personal tab → input/raw/personal.csv

Spatial input: - Leeds boundary shapefile → input/raw/boundary.shp

1.2 Script

process_decisions.R

1.3 Description

This script:

  • Loads raw DECISIONS trip and personal data
  • Converts origin and destination coordinates from Well-Known Binary (WKB) to British National Grid
  • Filters invalid trips and trips outside the Leeds boundary
  • Extracts derived trip attributes required for later analysis
  • Saves the processed dataset to input/processed/trips.csv

2 Compute R5 routing attributes

2.1 Requirements

The directory input/raw/r5r/ must contain:

  • OpenStreetMap extract (.pbf)
  • Bus GTFS feed (.zip)
  • Rail GTFS feed (.zip)
  • Elevation raster (.tif)

The elevation raster can be generated using get_elevation.R

2.2 Script

get_R5_attributes.R

2.3 Description

This script:

  • Loads processed DECISIONS trip data
  • Builds a multimodal R5 transport network
  • Computes distance and time estimates for each trip
  • Merges results into a wide format dataset (one row per trip)
  • Saves output to input/processed/R5_attributes.csv

3 Compute dodgr/OSMRN (DO) attributes

3.1 Requirements

The directory input/raw/ must contain the OS MRN geopackage osmrn.gpkg

3.2 Script

get_DO_attributes.R

3.3 Description

This script:

  • Loads the losdos package
  • Buffers the Leeds boundary by 2 km and saves it as input/raw/boundary.gpkg
  • Loads DECISIONS trips and formats required columns
  • Filters out public transport trips, as the OS MRN is limited to walking, cycling, and driving
  • Computes distance and time using osmrn_trip_attributes() from losdos
  • Saves outputs to input/processed/DO_attributes.csv

4 Compute Google Routes (GR) attributes

4.1 Requirements

The directory input/raw/ must contain a valid Google Maps API key in api_key.txt

4.2 Script

get_GR_attributes.R

4.3 Description

This script:

  • Defines a wrapper function for calls to the Google Routes API
  • Computes distance and duration estimates for each trip and mode
  • Handles public transport modes separately:
    • transitPreferences set to BUS or RAIL
    • Step-level vehicle types are extracted from routes.legs.steps.transitDetails.transitLine.vehicle.type
    • If the identified route for a bus trip contains either no bus stages or at least one rail stage, then it is marked as unroutable, as this analysis is limited to unimodal journeys
    • Likewise, if the identified route for a rail trip contains either no rail stages or at least one bus stage, then it is marked as unroutable
    • Alongside total distance and time, the waiting, in-vehicle and transfer/access times are also extracted, plus the total number of transfers
  • Merges all mode-specific results into a single dataset
  • Saves outputs to input/processed/GR_attributes.csv

6 Compare routing accuracy

6.1 Script

compare_accuracy.R

6.2 Description

This script:

  • Converts trip attributes into a long-format comparative dataset
  • Filters estimates to those matching each trip’s observed mode choice
  • Estimates parking time adjustments using quantile regression:
    • rq function from quantreg package used
    • Outputs saved to input/processed/parking_times.csv
  • Computes accuracy metrics:
    • ratio: recorded/(estimated + parking)
    • log-ratio
    • absolute log-ratio
  • Produces:
    • comparative scatter plots
    • kernel density plots of log-ratios
  • Conducts statistical tests:
    • Friedman tests for walk, bike, and car
    • Pairwise Wilcoxon tests for all modes

7 Prepare revealed-preference dataset

7.1 Script

process_rp.R

7.2 Description

This script:

  • Defines availability variables for bus and rail by each routing method
  • Constructs the choice variable conditional on observed mode
  • Sets taxi times equal to car times prior to parking adjustment
  • Applies parking adjustments to bicycle and car times
  • Defines travel costs:
    • Car: £0.45 per mile
    • Taxi: Leeds City Council 2026 fares, discounted to 2016/17 prices
    • Bus: 2016 First Bus single fares
    • Rail: assumed equal to bus fares
  • Saves the RP dataset to input/processed/trips_rp.csv

8 Estimate choice models

8.1 Scripts

  • choice_models_GR.R
  • choice_models_R5.R
  • choice_models_DO.R

8.2 Description

Each script:

  • Estimates base model and perturbation models using the apollo package
  • Applies mode availability constraints
  • Replaces missing perturbation estimates with base estimates, collapsing the corresponding perturbation terms to zero
  • Estimates models by fixing the appropriate gamma and delta parameters
  • Collects key outputs, estimates and robust standard errors into a summary table
  • Saves:
    • Model objects to output/choicemodels/
    • Summary tables to output/choicemodels/<base>_output.csv
